1. 概觀

SanJiaoIM 是 macOS 26 Tahoe 以上的三角編號法（三角編號，Three-Corner Code）輸入法。編碼規則源自胡立人、張源渭、黃克東（美國王安電腦公司）提出的三位數字碼檢字法，經由 6 位數字（部首首角 + 左下角 + 右下角等三角特徵）精確定位一個漢字。

專案特性：

  • 純 Swift 6.2 + InputMethodKit 實作，無第三方執行時依賴
  • 碼表涵蓋 ~32,900 字（Big5 + GB + CJK-B 擴展）
  • 使用者字頻學習（本機儲存，無遙測）
  • 分層候選顯示：Big5 常用字優先，罕字墊底
  • MIT 授權開源
碼表來源
chinese-opendesktop/cin-tables 的 3corner.cin（Public Domain）。

2. 系統需求

項目需求
macOS26 Tahoe 或更新
Xcode16 或更新（含 Swift 6.2 toolchain）
Homebrew已安裝
xcodegen2.38 或更新（若無，安裝指令：brew install xcodegen）
磁碟空間約 200 MB（含建置中間產物）
權限可寫入 ~/Library/Input Methods/
macOS 26 限制
本版本使用 Tahoe-only API，無法在 macOS 25 或更早版本運行。若需支援舊版系統，請自行 fork 並調整 deploymentTarget。

3. 快速安裝（推薦）

適合第一次安裝的使用者。整個流程約 2 分鐘（第一次建置多花 30 秒編譯）。

  1. 取得原始碼

    git clone https://github.com/<your-github>/SanJiaoIM.git
cd SanJiaoIM

    若你是本機直接開發（此倉庫已 clone 完成），跳過。

  2. 確認 xcodegen 已安裝

    which xcodegen || brew install xcodegen

  3. 執行一鍵安裝腳本

    ./scripts/install-dev.sh

    此腳本會：

    1. 從 Vendor/3corner.cin 建立 App/Resources/Lexicon.bin（碼表約 830 KB）
    2. 以 Debug 模式建置 SanJiaoIM.app
    3. 將 .app 複製到 ~/Library/Input Methods/SanJiaoIM.app（若已存在會先移除）

    成功訊息：Installed to /Users/you/Library/Input Methods/SanJiaoIM.app

  4. 登出並重新登入

    macOS 需要重啟 session 才會掃描新的輸入法 bundle。按 ⌘ Q → 選「登出」。

    替代方案（不登出）
    執行 killall Dock 通常可逼迫 HIToolbox 重新掃描，但不保證成功。登出最穩。

  5. 啟用輸入法

    見下一節「5. 啟用輸入法」。

4. 逐步手動安裝

若快速安裝失敗，或你想掌握每個步驟，依以下流程逐步操作。

4.1 建置碼表

./scripts/build-lexicon.sh

預期輸出：

Build of product 'sanjiao-builder' complete! (3.4s)
wrote 32889 entries → /path/to/repo/App/Resources/Lexicon.bin
-rw-r--r--  1 user  staff   830K Apr 20 12:00 Lexicon.bin

4.2 產生 Xcode project

xcodegen generate

此步會依 project.yml 重建 SanJiaoIM.xcodeproj。任何對 project.yml 或新增的 Swift 檔案都需要 re-run。

4.3 跑測試（選用但推薦）

xcodebuild -project SanJiaoIM.xcodeproj -scheme SanJiaoIM test

預期：** TEST SUCCEEDED **。此步會跑 smoke test 確認 Lexicon.bin 已正確 bundle 進 .app。

4.4 建置 Debug 版 app

xcodebuild -project SanJiaoIM.xcodeproj -scheme SanJiaoIM \
    -configuration Debug -derivedDataPath build build

完成後產物位於：

build/Build/Products/Debug/SanJiaoIM.app

4.5 複製到輸入法目錄

DEST="$HOME/Library/Input Methods/SanJiaoIM.app"
rm -rf "$DEST"   # 若已存在
cp -R build/Build/Products/Debug/SanJiaoIM.app "$DEST"

4.6 登出 / 登入

新輸入法 bundle 才會被 HIToolbox 掃描。

5. 啟用輸入法

  1. 打開系統設定

    ⌘ , 於桌面不通用 — 透過選單列「 → 系統設定」。

  2. 前往「鍵盤 → 輸入來源」

    左側導覽：「鍵盤」。右側找「輸入來源」→ 點「編輯…」或「加入」。

  3. 選擇「繁體中文 → SanJiaoIM」

    語言選單下拉，找到「繁體中文」（或「中文 — 繁體」）分類，SanJiaoIM 應顯示於其中。勾選並按「加入」。

    看不到 SanJiaoIM？
    請確認你已登出重新登入。若仍看不到，執行： 
    ls ~/Library/Input\ Methods/SanJiaoIM.app
    若檔案存在但未列出，試執行 killall Dock SystemUIServer，或重開機。

  4. 切換到 SanJiaoIM

    按輸入法切換快捷鍵（預設 Ctrl Space 或 Caps Lock，依你的設定）。選單列右上角的輸入法圖示應切至 SanJiaoIM。

  5. 處理 Gatekeeper 警告（僅首次）

    由於是 ad-hoc 簽章（非 Apple 開發者 ID），macOS 會阻擋第一次執行。

    1. 打開任一編輯視窗（Safari 搜尋列即可），切到 SanJiaoIM，試打任意按鍵。
    2. 右上角可能跳出「SanJiaoIM 已封鎖」提示。
    3. 前往「系統設定 → 隱私權與安全性」，捲到最下方，按「仍要打開」。
    4. 再試一次輸入。
    指令列替代方案
    若 GUI 批准失敗，可先移除隔離屬性： 
    xattr -rd com.apple.quarantine ~/Library/Input\ Methods/SanJiaoIM.app
    然後登出 / 登入。

6. 基本使用

切到 SanJiaoIM 後，在任何文字輸入框內：

按鍵作用
0 – 9輸入三角編號碼（最多 6 位）
Space以當前 buffer 查字並送出第一個候選
Enter右側補 0 至 6 位再查字（相當於前綴查詢）
1 – 9 / 0候選窗中選第 N 個（0 代表第 10 個）
. / ,候選視窗翻頁（前進 / 後退）
Backspace刪一位碼；若 buffer 變空則退出輸入模式
Esc取消輸入，回到原狀
a – z混打英文字母：正在組字時會先丟棄 buffer 再送出英文字母

6.1 試打範例

讀字「一」，三角編號為 100301：

  1. 依序按 1 0 0 3 0 1 → 第 6 位按下時自動彈出候選視窗，第一個就是「一」
  2. 按 Space 或 1 送出

若只記得前 4 位 1003：

  1. 按 1 0 0 3
  2. 按 Enter（Enter 會把 buffer 補 0 至 100300 後前綴查詢 1003* 的所有字）
  3. 在候選窗翻頁找到目標字，按對應數字鍵

6.2 字頻學習

每次從候選窗選字，系統會記錄該字使用次數與最後使用時間，下次同樣碼的候選排序會自動將常用字往前調。

資料儲存於：

~/Library/Application Support/SanJiaoIM/freq.json

純本機 JSON 檔，可 cat 檢視、備份、或搬到另一台電腦。

6.3 偏好設定 / 清除學習紀錄

  1. 選單列右上角輸入法圖示 → 下拉選單
  2. 選「偏好設定…」
  3. 按「清除學習紀錄」按鈕 → 確認

7. 疑難排解

7.1 SanJiaoIM 不在輸入來源清單

  • 確認 ~/Library/Input Methods/SanJiaoIM.app 存在：ls -ld ~/Library/Input\ Methods/SanJiaoIM.app
  • 確認已登出並重新登入（不是只關閉系統設定視窗）
  • 嘗試 killall Dock SystemUIServer cfprefsd
  • 檢查 app 是否被隔離：xattr ~/Library/Input\ Methods/SanJiaoIM.app，若列出 com.apple.quarantine 則執行 xattr -rd com.apple.quarantine ~/Library/Input\ Methods/SanJiaoIM.app

7.2 選單列出現紅色警告「⚠︎」

表示啟動時找不到 Lexicon.bin。原因通常是：

  • 你用了 ad-hoc 建置但跳過了 ./scripts/build-lexicon.sh
  • App/Resources/Lexicon.bin 不在 .app bundle 內

解法：回到倉庫執行

./scripts/build-lexicon.sh
./scripts/install-dev.sh

7.3 輸入時沒有候選窗

  • 確認已切到 SanJiaoIM（選單列圖示）
  • 按 Space 主動觸發（有時自動彈出會因 focus 切換而延遲）
  • 在 Console.app 過濾 subsystem:com.sanjiaoim.app 檢查是否有錯誤日誌

7.4 某些 app 內無法使用（例如 Electron app）

Electron / Chrome 類 app 的輸入法整合偶有相容性問題，現象可能是候選窗不顯示、或插入字到錯誤位置。目前 v0.1 已知限制，見 docs/manual-test-checklist.md。已驗證可正常使用的 app：Safari、Pages、Xcode、Terminal、TextEdit。

7.5 freq.json 損毀

若 freq.json 內容變成無效 JSON（手動亂編輯、磁碟損毀等），SanJiaoIM 啟動時會自動：

  1. 將損毀檔移至 ~/Library/Application Support/SanJiaoIM/freq.json.corrupt.bak
  2. 建立全新空白 freq.json

不需手動處理，但字頻學習會從零開始。

7.6 檢查日誌

log stream --predicate 'subsystem == "com.sanjiaoim.app"' --level debug

分類：core（carga 與查字）、imkit（IMKit 整合）、freq（字頻）。

8. 解除安裝

  1. 從輸入來源移除

    系統設定 → 鍵盤 → 輸入來源 → 選 SanJiaoIM → 按「−」。

  2. 刪除 .app bundle

    rm -rf ~/Library/Input\ Methods/SanJiaoIM.app

  3. （選用）刪除使用者資料

    rm -rf ~/Library/Application\ Support/SanJiaoIM

    包含字頻學習紀錄與所有備份。若你打算之後重裝，可保留這個目錄。

  4. 登出重新登入

    確保 HIToolbox 重新掃描、清除記憶中的 bundle 引用。

9. 開發者資訊

9.1 測試

# 核心套件單元測試（28 tests）
cd Packages/SanJiaoCore && swift test

# 建置器單元測試（16 tests）
cd Tools/sanjiao-builder && swift test

# App smoke test（驗證 Lexicon.bin bundle）
xcodebuild -project SanJiaoIM.xcodeproj -scheme SanJiaoIM test

9.2 重建 Xcode project

修改 project.yml、新增 Swift 檔、調整 Info.plist 後：

xcodegen generate

9.3 升級碼表

上游 chinese-opendesktop/cin-tables 更新時：

curl -sL https://raw.githubusercontent.com/chinese-opendesktop/cin-tables/master/3corner.cin \
  -o Vendor/3corner.cin
# 更新 Vendor/3corner.cin.SOURCE 裡的 commit SHA
./scripts/build-lexicon.sh
./scripts/install-dev.sh

9.4 重新產生 Big5 level-1 表

python3 Tools/sanjiao-builder/scripts/generate-big5-level1.py \
  > Tools/sanjiao-builder/Sources/SanJiaoBuilder/Big5Level1Table.swift

9.5 除錯技巧

  • IMKit 偵錯：log stream --predicate 'subsystem == "com.sanjiaoim.app"'
  • 觀察 Composer 狀態：在 SanJiaoInputController.handle 加 log.debug("state=\(c.state)")
  • 強制重載輸入法：終端機執行 killall SanJiaoIM 2>/dev/null; true（IMKit 會自動重啟）

9.6 回報 bug

請附上：

  • macOS 版本（⌘ → 「關於這台 Mac」）
  • 可重現的步驟
  • Console.app 匯出的日誌（過濾 com.sanjiaoim.app）
  • 若涉及字頻：freq.json 前幾行（head -20）
